Development Report

A reflection on my web development journey (325 words)

My Learning Journey

When I first started this module, I had very little experience with HTML and CSS. The initial weeks were challenging as I tried to understand the box model, positioning, and how to structure a webpage properly. I remember spending hours trying to center a div - a classic beginner struggle! However, through persistence and practice, I gradually built my understanding.

One of my biggest breakthroughs was understanding CSS Grid. Initially, I was using float and position: absolute for everything, which made my layouts fragile. When I discovered grid, everything clicked. I could create complex, responsive layouts with just a few lines of code. The projects page of this portfolio uses a masonry-style grid that demonstrates this learning.

Design Decisions

Color Scheme: I chose a dark theme with #1a1a2e for navigation and #e94560 as the accent color. This combination creates a professional, modern look that stands out. The dark background reduces eye strain, while the coral accent draws attention to important elements.

Typography: I selected 'Poppins' for headings and 'Open Sans' for body text. Poppins gives a modern, geometric feel while Open Sans ensures readability for longer text passages. Both are web-safe fonts via Google Fonts.

Layout Inspiration: My home page layout was inspired by modern portfolio sites on Awwwards and Dribbble. I wanted a hero section that immediately introduces who I am, followed by skills and vision sections that tell my story.

Technical Implementation

CSS Grid vs Flexbox: I used CSS Grid for main page layouts (home page hero section, skills grid, projects grid) because grid is two-dimensional and perfect for overall page structure. Flexbox was used for navigation menus and aligning icons within cards where content flows in one direction.

Responsive Design: I used a mobile-first approach with media queries at 768px. The hamburger menu appears only on mobile and was implemented with JavaScript toggling. On mobile, all sections stack vertically for optimal viewing.

Animations: I added subtle hover animations on skill items and project cards. The skill icons scale up slightly on hover, and project cards have a smooth shadow transition. These micro-interactions improve user experience without being distracting.
